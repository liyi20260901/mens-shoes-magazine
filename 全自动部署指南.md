# 全自动部署指南：每天自动生成男鞋电子杂志

## 目标
配置完成后，系统每天自动：
1. 抓取品牌男鞋官网数据
2. 生成电子杂志HTML
3. 部署到GitHub Pages
4. 获得可分享的链接

**全程无需手动操作**。

---

## 第一步：创建GitHub仓库（一次性）

1. 登录GitHub → 点击右上角 **+** → **New repository**
2. Repository name 填：`mens-shoes-magazine`
3. 选择 **Public**（GitHub Pages免费版需要Public）
4. 勾选 **Add a README file**
5. 点击 **Create repository**

---

## 第二步：上传初始文件（一次性）

在仓库页面：
1. 点击 **Add file** → **Upload files**
2. 把这三个文件拖进去：
   - `batch_scraper.py`（批量抓取脚本）
   - `magazine_template.html`（杂志模板）
   - `requirements.txt`（Python依赖，见下方）
3. 点击 **Commit changes**

### requirements.txt 内容：
```
requests>=2.28.0
beautifulsoup4>=4.11.0
```

---

## 第三步：配置GitHub Actions（一次性）

在仓库页面：
1. 点击 **Actions** 标签
2. 点击 **set up a workflow yourself**（或 **New workflow**）
3. 文件名填：`daily-magazine.yml`
4. 把下方完整YAML贴进去
5. 点击 **Commit changes**

### `.github/workflows/daily-magazine.yml` 完整内容：

```yaml
name: Daily Men's Shoes Magazine

on:
  schedule:
    # 每天北京时间早上8点运行（UTC 0点）
    - cron: '0 0 * * *'
  # 也支持手动触发（点一下按钮就跑）
  workflow_dispatch:

permissions:
  contents: write
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
      # 1. 检出代码
      - name: Checkout
        uses: actions/checkout@v4
      
      # 2. 设置Python环境
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      # 3. 安装依赖
      - name: Install dependencies
        run: |
          pip install requests beautifulsoup4
      
      # 4. 运行抓取脚本
      - name: Scrape products
        run: |
          python batch_scraper.py
        # 超时10分钟
        timeout-minutes: 10
      
      # 5. 生成杂志HTML
      - name: Generate magazine
        run: |
          python generate_magazine.py
        timeout-minutes: 5
      
      # 6. 上传生成的文件到 artifact
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './output'
      
      # 7. 部署到GitHub Pages
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4
```

---

## 第四步：开GitHub Pages（一次性）

1. 仓库页面 → **Settings** → 左侧 **Pages**
2. Source 选择 **GitHub Actions**
3. 点击 Save

def generate_magazine():
    output_dir = './output'
    os.makedirs(output_dir, exist_ok=True)
    
    # 查找所有品牌JSON文件
    json_files = glob.glob(f'{output_dir}/*_products.json')
    if not json_files:
        print("⚠️ No product data found, generating demo issue")
        return generate_demo_issue()
    
    # 读取所有品牌数据
    all_brands = []
    for jf in json_files:
        with open(jf, 'r', encoding='utf-8') as f:
            data = json.load(f)
            all_brands.append(data)
    
    # 选择本期主打品牌（轮询制）
    issue_number = datetime.now().timetuple().tm_yday  # 用一年中的第几天作为期号
    featured_idx = issue_number % len(all_brands)
    featured = all_brands[featured_idx]
    
    # 生成HTML
    html = build_html(featured, issue_number, len(all_brands))
    
    # 保存
    today = datetime.now().strftime('%Y%m%d')
    html_path = f'{output_dir}/index.html'
    with open(html_path, 'w', encoding='utf-8') as f:
        f.write(html)
    
    # 同时保存历史版本
    history_path = f'{output_dir}/issue_{today}.html'
    with open(history_path, 'w', encoding='utf-8') as f:
        f.write(html)
    
    print(f"✅ Generated: {html_path}")
    print(f"✅ History: {history_path}")
    return html_path

## 常见问题

**Q: 抓取失败怎么办？**
- GitHub Actions会自动重试，连续失败3天会发邮件通知你
- 你可以在Actions日志里看到具体错误

**Q: 想换品牌或加品牌？**
- 修改 `batch_scraper.py` 里的 `BRANDS_CONFIG` 列表
- 推送到仓库，第二天自动生效

**Q: 想改杂志风格？**
- 修改 `generate_magazine.py` 里的HTML/CSS
- 推送到仓库，第二天自动生效
